# 竞品对比：WebAISearch vs 市面搜索方案

> 当前 Agent 获取外部信息的三种路径：API 搜索、Agent 内置搜索、浏览器自动化搜索。
> 本文从开发者日常使用场景出发，对比三类方案的实际体感。

---

## 一、市场现状概览

### 类型 A：API 搜索服务（付费）

| 服务 | 免费额度 | 付费门槛 | 特点 |
|------|---------|---------|------|
| Tavily | 1000次/月 | .005/次起 | AI-native，专为 Agent 设计 |
| SerpAPI | 无 | /月起 | 传统 SERP，兼容性广 |
| Brave Search | 2000次/月 | /1000次 | 自有索引，隐私友好 |
| Firecrawl | 500次/月 | /月起 | 搜索+内容提取一体 |
| Exa | 1000次/月 | .005/次起 | 神经网络搜索，语义理解 |

### 类型 B：Agent 内置搜索（免费，但受限）

| Agent | 搜索能力 | 实现方式 | 局限 |
|------|------|---------|------|
| Claude Code | 内置 web_search | 直接调用 | 仅返回网页摘要，无 AI 分析 |
| Codex CLI | Tavily MCP / web_search | MCP 协议 | 需配置 MCP，或依赖内置引擎 |
| Cursor | 内置搜索 | 平台后端 | 不可切换搜索引擎 |
| Gemini CLI | Google 搜索 | 原生集成 | 仅 Google，无多平台 |

### 类型 C：浏览器自动化搜索（本 Skill）

| 方案 | 费用 | 依赖 | 特点 |
|------|------|------|------|
| **WebAISearch** |  | 浏览器 + Playwright | 操控 AI 平台的 Web 界面，零封号风险 |

## 二、核心差异：“调接口” vs “长了手”

三种方案的本质区别：

`
API 搜索：  Agent → API Key → 搜索引擎 → 网页摘要 → Agent
Agent 内置： Agent → 内置工具 → 搜索后端 → 结果
WebAISearch： Agent → CDP → 浏览器 → AI平台Web界面 → AI分析后的结果
`

**API 搜索是“买数据”，Agent 内置是“用工具”，WebAISearch 是“用平台”——直接利用 AI 平台的搜索+推理能力。**

## 三、六维对比

### 1. 费用

| | API 搜索 | Agent 内置 | WebAISearch |
|---|---------|------------|-------------|
| 月费用 | ~+ | （包含在 Agent 订阅中） | **** |

> **一个开发者每天 20 次搜索，Tavily 超出免费额后每月约 .5。WebAISearch：。**

### 2. 搜索质量

| | API 搜索 | Agent 内置 | WebAISearch |
|---|---------|------------|-------------|
| 结果形式 | 网页摘要/标题/URL | 网页摘要 | **AI 分析后的结构化回复** |
| 推理链 | 无 | 无 | **有（DeepSeek CoT）** |
| 多方案对比 | Agent 自己做 | Agent 自己做 | **AI 平台做好再给你** |
| 信息深度 | 取决于索引覆盖 | 取决于后端 | **取决于你选的 AI 平台** |

> **API 给你原料，你自己做菜。WebAISearch 给你做好的菜。**

### 3. 平台自由度

| | API 搜索 | Agent 内置 | WebAISearch |
|---|---------|------------|-------------|
| 切换搜索引擎 | 换 API key | 不可切换 | **换 URL 即可** |
| 支持中文平台 | 有限 | 有限 | **DeepSeek/ChatGPT/Gemini 随意换** |
| 新增平台成本 | 新 API 对接 | Agent 更新 | **LLM 自动分析 DOM 定型** |

### 4. Agent 兼容性

| | API 搜索 | Agent 内置 | WebAISearch |

| | API 搜索 | Agent 内置 | WebAISearch |

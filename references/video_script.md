# 视频文案：我给 AI 加了个能自己用浏览器搜索的东西

> 风格：一个开发者随便聊聊自己做的小工具，不是教你什么，就是分享一下。

---

你们写代码的时候有没有这种情况：

报了个错，不知道啥意思，复制错误信息，切浏览器，打开搜索，粘贴，等结果，点开三四个链接，看完发现不是那个意思，又换个搜索词，又看。搞完半小时过去了，代码一行没改。

我就想，这事能不能让 AI 自己去干？

然后我就去看了下现在的解决方案。一种是买 API，Tavily、SerpAPI 这些，要注册要付费，搜一次几分钱，我懒得搞。还有一种是各个 AI 编程工具自带的搜索功能，免费，但它就是丢给你一堆网页摘要，还得自己点进去看。

那我就想啊，我平时自己遇到问题都是直接扔给 DeepSeek 或者 ChatGPT 的，它们回答得挺好的啊，而且它们本身就会自己去搜索、去分析，直接给我结论。干嘛还要一个中间商在那里转一手？

所以我就做了这个东西：让我的 AI 编程助手能自己打开浏览器，去 DeepSeek、ChatGPT、Gemini 这些地方帮我搜。搜完了自己把结果抓回来。我只要告诉它“这个报错帮我查一下”，后面的事我就不管了。

具体怎么实现的呢？其实就是用 Playwright 这个库去控制浏览器，通过 CDP 协议发指令。你可以理解为就是给你的浏览器写了个“自动化脚本”，但是这个脚本不是我手写的，是让 AI 自己看一下页面长什么样，然后自己生成的。所以哪怕平台改版了，再跑一次就行，不用我去改代码。

提取结果这块也挺有意思的。我没有用什么复杂的 DOM 解析，而是在发送的时候就在 prompt 里面塞了一个唯一标记，比如 `[搜索主题：bug AttributeError@3f7a]`，然后告诉 AI “你回答的时候开头和结尾都放一个这个”。等它回复完了，我只要在页面上找这两个标记，中间的就是我要的东西。简单、不依赖页面结构，换哪个平台都一样。

一些使用上的细节：

它会自己判断你的问题是代码报错、技术选型、还是新闻查询，然后用不同的搜索策略。报错了就自动走 debug 模式，技术选型就走 code 模式，不用你跟它说“你要怎么搜”。

你可以设成固定对话，每次搜索不开新窗口，而是在同一个对话里继续。这样 AI 记得上一次你问了什么，不用每次都重新解释项目背景。

安装也很简单，就是一个 SKILL.md 丢到对应目录，然后跑一下 setup.py。国内的话它会自己走清华镜像源装依赖，不用担心 pip install 半天不动。

至于费用，完全免费。你用的是你自己的浏览器，访问的是免费的 AI 平台，不存在任何付费环节。而且因为它就是普通的浏览器操作，不是爬虫不是反代，没有封号风险。

项目开源的，GitHub 上随便拿。觉得有用的话可以试试，有问题直接留言。

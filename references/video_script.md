## 开场（前 15 秒钩子）

你有没有这种时候——

正在写代码，突然报错了。你把错误信息复制下来，切到浏览器，打开搜索，粘贴，等结果，点开几个链接，看完，找到了，再切回来改代码。然后又报错了，再来一遍。

这个过程重复三次，你一上午就过去了。

有没有办法让 AI 自己去搜？

现在市面上的方案是什么呢？ 买 API。Tavily、SerpAPI，充值、拿 key、配置，搜一次几分钱，一天下来也得好几块钱。还得担心超额。

那另一条路：让 AI 直接用你的浏览器。开网页、打字、等回复、把结果复制回来。就像你自己在操作一样，只不过是自动的。

这就是我做的东西——WebAISearch。一个完全免费的、开源的、给 AI 编程助手用的自动搜索 Skill。

## 为什么不用 API 搜索？

[对比表可以做成图卡效果]

现在市面上 AI 搜索主要两种：

第一种，买 API。Tavily 一个月免费 1000 次，超了就付费。SerpAPI 直接  起步。你一个开发者每天搜 20 次，一个月就得多花好几十块钱。而且 API 返回的是什么？ 网页摘要。一堆链接，你还得自己打开看。

第二种，Agent 内置搜索。比如 Claude Code 自带的 web_search。免费，但也是给你网页摘要。而且只能用它自己的搜索引擎，你没得选。

而这个 Skill 的思路完全不一样——

它不是去调 API，而是直接把 AI 平台的 Web 界面当成工具用。你去问 DeepSeek、问 ChatGPT、问 Gemini——它们本身就会搜索、就会分析、就会总结。干嘛还要自己看网页？

等于是白嫖了 DeepSeek 的搜索能力，还不用花一分钱。

## 核心亮点

[配合画面或动画]

**第一，完全免费。**不是“免费额度”，是真的一分钱不用。因为你用的是你自己的浏览器，访问的是免费的 AI 平台，没有任何中间商。

**第二，不怕封号。**它就是在真实操作你的浏览器，跟你自己打字没区别。不是反代、不是爬虫、不是模拟协议。就是普普通通的浏览器自动化。

**第三，什么 AI 助手都能用。**Claude Code、Codex、Cursor、Gemini CLI——只要支持 SKILL.md 标准，一个文件丢进去就能用。不需要每个工具单独配置。

**第四，搜索平台随便换。**今天用 DeepSeek，明天用 ChatGPT，后天用 Gemini。一行配置的事。同一个问题发三个平台，三方互校，谁在胡说你一目了然。

**第五，安装就是一句话。**没有 API key、没有注册、没有充值。而且国内用户自动走清华镜像源安装依赖，不用担心 pip install 卡半天。

## 一个有意思的技术点：“标记对”提取

[可以做个小动画说明]

这里面有个很巧的设计。

你想啊，你在 DeepSeek 里发了一个问题，它回答了。然后你又发了第二个问题。现在页面上有两段回答，你怎么精确提取第一次的结果？

传统办法是用 DOM 选择器——找页面上某个 CSS class。但这个办法很脆弱，平台一改版就挂了。

这个 Skill 用了一个更聪明的方法：在发送的时候，在 prompt 里埋一个唯一标记，比如「搜索主题：bug AttributeError@3f7a」。然后要求 AI 在回答的开头和结尾各放一个。提取的时候只要找这两个标记，中间的内容就是结果。

这个方法完全不依赖页面结构，任何 AI 平台都能用，永远不会因为平台改版而失效。

## 实际效果

[快速切换三个演示画面]

我们测了一下。同一个技术问题，分别发给 DeepSeek、ChatGPT 和 Gemini。

DeepSeek 最快，22 秒出结果。ChatGPT 34 秒。Gemini 慢一点，但也能用。

关键是，它们回复的不是一堆链接，而是已经分析好的结论。根因是什么、怎么修复、有没有替代方案。直接就能用。

而且如果你是固定模式，它会记住你的对话上下文。下一次搜索不用重新解释你在做什么项目。

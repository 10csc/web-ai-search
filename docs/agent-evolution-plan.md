# WebAISearch 核心目标：三级自进化 Agent

> 终极目标：不仅是"能搜索的脚本"，而是一个**能自我适应、自我调整、自我改进**的搜索研究者。
> 不仅要智（理解问题、分解任务），还要能（自主选择策略、动态调整行为）。

---

## 一、设计哲学

### 核心理念

```
不依赖"常理"。每个平台的行为随时可能变化——
  - DeepSeek 可能会限制快速模式
  - Kimi 可能调整思考/快速/Agent 的可用性
  - SCnet 可能换底层模型
  - Gemini 网页端加载速度可能变化

适应这些变化的不是"更新代码"，而是"自进化"。
```

### 三级进化关系

```
配置自进化（地基）
    ↓ 环境就绪后
决策自进化（大脑）
    ↓ 策略确定后
内容自进化（双手）
    ↓ 执行并反馈
    回到决策自进化（闭环）
```

### 触发原则：懒进化（Lazy Evolution）

只在**必要时**进化，不在正常运行时消耗资源：

| 进化类型 | 触发条件 |
|---------|---------|
| 配置自进化 | 出错时（连接失败、DOM 变化、新平台首次使用） |
| 决策自进化 | 近期多次出现显著评分差异（某平台持续输给另一平台） |
| 内容自进化 | 最终报告被评价为偏离方向（synthesis/judge 层判定） |

## 二、配置自进化（地基层）

### 目标
在核心理念不变的情况下，自动配置所需环境，包括出错时的重配置。

### 范围

| 子能力 | 当前状态 | 目标状态 |
|--------|---------|---------|
| **环境自举** | setup.py 手动执行 | CDP 端口自动扫描、venv 自动检测、依赖自动安装 |
| **平台定型** | generator.py 生成交互脚本 | DOM 变化时自动检测 → 自动重新定型 |
| **会话管理** | config.json 手动绑定 | 会话失效自动检测 → 自动创建新会话 → 自动绑定 |
| **错误恢复** | 手动排查 | CDP 重连、页面刷新、超时递增退避——全部自动化 |
| **新平台注册** | 手动添加 URL + 路由 | 用户提供 URL → 自动定型 → 自动注册到路由表 |

### 关键设计
- 配置自进化**只在出错时触发**（正常运行时零开销）
- 每次自修复后记录到 `profiles/_global.json` 的 `config_evolution_log`
- 同一错误连续 3 次自修复失败 → 告警用户，不无限循环

## 三、决策自进化（大脑层）

### 目标
用什么模型搜什么内容，由数据驱动——而非硬编码路由。

### 当前平台理解（2026-06）

| 平台 | 底层模型 | 核心优势 | 核心劣势 | 特殊能力 |
|------|---------|---------|---------|---------|
| **SCnet（超算）** | DeepSeek V4 Pro | V4 Pro 满血推理、可上传文件 | 搜索仅~10网页 | 关联网=文件上传模式 → 天然适合最终报告整合 |
| **DeepSeek** | DeepSeek V4 Flash+Think | DeepThink 深度推理、不限流 | 当前仅快速模式可用，分析能力受限 | 需决策：搜索 vs 专家模式？ |
| **Kimi** | Kimi K2.6 | 中文信息覆盖最强（微信/知乎/CSDN）、WideSearch 80.8% 榜首 | **速度最慢**（需独立轮询策略） | 需决策：思考 vs 快速 vs Agent 功能？ |
| **ChatGPT** | GPT-5.x（免费阉割版） | 总结归纳扎实 | **严格限流、次数极少** | 只能作备选整合 |
| **Gemini** | Gemini Flash | **加载速度最快**、Google 索引 | 深度不如前两者 | 英文搜索首选 |
| **秘塔** | 自研 | 知网/万方穿透、结构化大纲 | 通用领域弱 | 学术交叉验证 |
| **Perplexity** | 基础模型 | 英文检索精度高、引用规范 | 免费 ~5次/天 | 节约使用 |

### 决策维度（每平台 × 每次搜索）

不只选平台，还要选**模式**：

```
SCnet:     [搜索模式] / [文件上传模式（关联网）]
DeepSeek:  [快速模式] / [专家模式（DeepThink）]
Kimi:      [快速模式] / [思考模式] / [Agent 模式]
Gemini:    [标准搜索] / [深度模式（不可用，免费版）]
ChatGPT:   [基础搜索] / [深度研究（不可用，免费版）]
```

### 决策自进化循环

```
同一问题 → 发送到 N 个平台 → 提取结果 → 交叉评分
    ↓
评分数据积累（≥10 组对比后触发）:
  - 中复杂度代码问题: Kimi快速 > DeepSeek快速（近期 7/10 次）
    → 自动调整路由：该场景优先 Kimi
  - 中文研报: 秘塔 > Kimi（近期 6/8 次）
    → 自动调整路由：学术类优先秘塔
    ↓
更新 PLATFORM_ROUTES + 持久化到 profiles/_routing.json
```

### 关键设计
- 评分来源：最终 synthesis/judge 层对每个平台输出的评价
- 触发阈值：同一场景 ≥10 组对比 + 显著差异（>60% 倾向某一方）
- 衰减窗口：只看近 30 天数据（平台能力随时变化）
- **不影响当前搜索**：调整在后台进行，下次搜索才生效

## 四、内容自进化（双手层）

### 目标
自动化保证搜索内容的完整性和方向正确性。

### 三个子方向

#### 4.1 提取自进化（已实现）
- CoT 关键词自动发现 + 跨平台传播
- 页尾特征自动识别
- 每平台独立提取档案

#### 4.2 发送自进化（Prompt 引导）
- **问题**：同样的 prompt 格式，不同模型响应质量不同
- **进化方向**：根据模型返回内容的质量评分，逐渐学习如何引导模型
  - 例如：SCnet 在标记对之间容易放思考块 → prompt 增加"请将正式回复放在标记之后"
  - 例如：Kimi 回复容易缺少来源链接 → prompt 强调来源要求
- **触发时机**：最终整合被 judge 层判定为"信息不完整"或"偏离方向"

#### 4.3 方向性引导
- **问题**：搜索 Agent 可能被模型带偏（模型误解了用户意图）
- **检测**：synthesis/judge 层比较"用户原始问题"和"最终报告方向"的语义相似度
- **修正**：偏离 > 阈值 → 重新规划子问题 → 重新搜索
- **学习**：记录偏离模式 → 下次分解问题时避开该类错误

### 关键设计
- 内容自进化**只在最终整合/裁判阶段反馈**时触发
- 修改的是 prompt 模板（`prompt_builder.py` 的策略参数），不是实时 prompt
- 每个平台的 prompt 策略独立进化

## 五、Judge 层（裁判——闭环的关键缺失）

三个自进化都需要一个**裁判**来提供反馈信号。当前系统缺失这一层。

### 职责
1. 对每个平台的输出打分（相关性、完整性、可信度）
2. 判断最终报告是否偏离用户原始意图
3. 输出结构化反馈 → 驱动三级自进化

### 实现方式
不引入额外 LLM 调用。在 synthesis 阶段，用一个**评分 prompt** 发给负责最终整合的平台（SCnet 或 Kimi），让它同时做两件事：
1. 整合报告
2. 输出评分（JSON 块，用标记包裹）

```
[评分]
{"overall_score": 8.2, "direction_match": 0.9, "sources": [
  {"platform": "deepseek", "score": 7.5, "issues": ["缺少来源链接"]},
  {"platform": "kimi", "score": 8.8, "issues": []}
], "replan_needed": false}
[/评分]
```

## 六、实现优先级

| 优先级 | 任务 | 对应进化 | 理由 |
|--------|------|---------|------|
| **P0** | Judge 层（评分机制） | 全部 | 无裁判则无反馈，无反馈则无进化 |
| **P1** | 平台模式选择器 | 决策 | DeepSeek 专家/搜索、Kimi 思考/快速——当前最大浪费 |
| **P1** | Prompt 策略档案（每平台） | 内容 | 各平台回复质量参差不齐，需要引导 |
| **P2** | 交叉评分数据积累 | 决策 | 需要足够对比数据才能调整路由 |
| **P2** | 方向偏离检测 + 重规划 | 内容 | 最终报告质量兜底 |
| **P3** | 配置自动恢复 | 配置 | CDP 重连、页面刷新——已有部分基础 |
| **P3** | 新平台自动注册 | 配置 | 降低用户接入新平台的门槛 |

## 七、补充：我不在三个维度里但应该在的

### 7.1 时间感知
Kimi 高峰期（工作日 9-11、14-17）稳定性下降 → 决策自进化应学到的模式。路由决策需要考虑"当前时段是否适合用 Kimi"。

### 7.2 进化预算
每次进化（写文件、跑 LLM 分类、重新定型）不是免费的。需要进化预算机制：
- 每小时最多触发 2 次内容自进化
- 每天最多触发 1 次决策自进化
- 配置自进化只在出错时触发，不限次数

### 7.3 进化回退
自进化可能学错（一次偶然的失败不代表趋势）。每次进化生成新策略时，保留旧策略的备份。新策略连续 3 次表现更差 → 自动回退。
